# YML
YML 파일은 개발 및 운영 환경에서 매우 유용한 도구
.yml,.yaml은 YAML을 나타내는 `파일확장자`입니다.
- YML은 주로 구성파일이나 데이터의 저장 및 전달에 사용된다.
- 사람이 읽기 쉽고,다른 프로그래밍 언어와도 호환성이 우수함
- 들여쓰기의 강제성 : YML은 가독성이 좋은대신, 들여쓰기를 꼭해야한다.(들여쓰기 수준도 같아야함)
- 콜론 뒤의 공백 : YML은 반드시 콜론뒤에 한칸을 띄어야 한다.
```
예시 ) key: value
```
- 여러 모듈이나 복잡한 설정을 가졌다면 YML이 구조를 명확히 표현하기 좋음
# Properties
properties은 응용 프로그램의 구성 가능한 <B> 파라미터</B>들을 저장하기위해 `자바 관련 기술을 사용하는 파일들을 위한 파일 확장`

- 파라미터 : 시스템이나 모델의 동작과 결과를 결정하는 내부의 설정 값(수치)
-> 예) 에어컨을 쓰면 온도를 어떻게 할지 바람 세기를 어떻게 할지 정함,
여기서 `온도`와 `바람세기`가 <B>파라미터</B>


 - Properties는 간결하고 직관적이다.

# Properties와 YML 확장자의 차이점
|구분| YML | Properties|
|---|----|----|
|구조|계층적구조(트리형태)|평면적 구조(한줄씩)|
|가독성|관계가 명확(복잡한 성정에 좋음)|단순 직관적(소규모에 좋음)|
장점|깔끔함(중복키 작성 줄임 )|배우기 쉽고 문법 오류 적음|
