# ERD(Entity-Relationship Diagram)
ERD는 프로젝트 할때 데이터베이스의 구조와 관계를 그래픽적으로 표현한 도구이다.

- ERD는 데이터베이스의 논리적 구조를 파악하는데 매우 유용하고, 데이터 설계시 커뮤니케이션 도구로 사용된다



# ERD 구성하는 요소
## 1 . Entity (개체)
엔티티는 데이터베이스에서 표현하려는 정부의 주요 객체의 개념을 나타낸다

## 2 . Attribute (속성)
엔티티의 세부 정보를 나타낸다
```
예시) 사용자 엔티티의 속성은 이름/이메일/전화번호
```

## 3 . Relationship (관계)
두 엔티티의 관계를 나타낸다
```
예시) 사용자와 주문의 관계 -> 주문하기
ERD에서는 라인과 관계의 이름을 표시한다, 연관된 엔티티관에 그림
```

## 4 . Cardinality (대응수)
엔티티 간 관계의 수를 나타낸다
```
예시) 사용자 여러 사용자에
```



| 요소 | 설명 | 형태(표기) |
|------|------|-----------|
| entity(객체) | 시스템이 기억해야할 대상 | 네모 |
| Attribute(속성) | 엔티티가 가진 상세 정보/특징 | 텍스트 리스트 |
| Relationship(관계) | 엔티티와 엔티티 간의 연결 고리 | 선|


# PK (Primary Key / 기본키)
엔티티(테이블)에서 **각 행(레코드)을 유일하게 식별할 수 있는 고유한 속성**이다.

- **특징:** 중복된 값을 가질 수 없으며(Unique), 비어있을 수 없다(Not Null).
- **ERD 표기:** 일반적으로 속성 이름 옆에 `PK`라고 표시하거나, 열쇠 모양 아이콘, 또는 속성 이름 밑에 **밑줄**을 그어 표현한다.
- *예시:* 사용자 엔티티의 `사용자_ID`, 상품 엔티티의 `상품_코드`

# FK (Foreign Key / 외래키)
한 엔티티의 속성 중 **다른 엔티티의 PK를 참조하는 속성**으로, 엔티티 간의 **관계(Relationship)를 맺어주는 역할**을 한다.

- **특징:** 참조하는 대상 테이블의 PK에 존재하는 값만 가질 수 있다(참조 무결성).
- **ERD 표기:** 속성 이름 옆에 `FK`라고 표시하며, 선을 연결할 때 어느 테이블에서 온 것인지 관계선으로 나타낸다.
- *예시:* 주문 엔티티에 있는 `사용자_ID` (어떤 사용자가 주문했는지 알기 위해 사용자 테이블의 PK를 가져와 FK로 사용)
