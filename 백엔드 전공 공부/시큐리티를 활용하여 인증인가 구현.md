# Security
스프링 시큐리티는 스프링 기반 애플리케이션의 `보안`을 담당하는 프레임워크

<b>spring security</b> : `인증,권한 관리 그리고 데이터 보호 기능을 포함`하여 웹 개발 과정에서 필수적인 `사용자 관리 기능`을 구현하는데 도움을 주는 spring 프레임워크

## spring security 사용이유
- spring security가 spring의 생태계에 보안에 필요한 기능을 제공

- spring security가 인증,인가 부분을 개발하기 편하게 하는 기능을 제공해줌(개발 작업 효율 상승)

# spring security 아키텍처(구조/디자인 패턴)
![alt text](image.png)
-> spring에서 프레임워크나 라이브러리를 사용했을때, 작동되는 구조를 표현한 그림

- 붉은 박스 : spring 프레임 워크에서 spring security가 적용되는 부분

```
1. 사용자의 요청(아이디/비번)이 서버로 들어옴

2. Authotication Filter가 요청을 가로채고 인증용 토큰 생성

3. Authotication Manger로 요청을 위임합니다.(누가 처리할수 있는지 찾음)

4. Authotication Manager는 등록된 Authotication Provider를 조회하며 인증을 요구합니다.

5. Authotication Provider가 실제 데이터를 조회하여 UserDetails 결과를 돌려줍니다.

6. 결과는 SecurityContextHolder에 저장이 되어 저장된 유저정보를 Spring Controller에서 사용할 수 있게 됩니다.
```
